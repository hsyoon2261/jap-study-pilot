# 학습 앱 구성도 (1개월+ 운영 기준 설계)

> 2026-07-17 설계. 원칙: 홈은 "오늘 할 일"만 보여주는 작업대, 과거는 기록 화면, 콘텐츠는 데이터(덱·학습지 JSON)로 추가 — UI 코드는 안 늘린다.

## 화면 구성

```
┌─ index.html (홈 = 오늘의 작업대) ───────────────────┐
│ 1. 오늘의 세트 (sets.json — 매일 5개, 완료는 학습자가) │
│ 2. 복습 대기 (SRS due 항목 — 쌓이면 여기 뜸, 원클릭 드릴)│
│ 3. 현황 요약 (연속일·오늘·누적) + "기록 전체 →"       │
│ 4. 학습 자료 (오늘의 학습지 / 가나 학습표)            │
│ 5. 자유 연습 (덱 목록 → 범위·모드 직접 골라 드릴)      │
└──────────────────────────────────────────────┘
   ├─ sheets.html  학습지 목록 (사이드바 진입점)
   ├─ sheet.html   학습지 상세 (?day=N, 파라미터 없으면 최신 — 홈 바로가기 진입점)
   ├─ chart.html   가나 학습표 (상설 레퍼런스)
   ├─ browse.html  덱 열람 (?deck= 파라미터로 아무 덱이나)
   └─ history.html 기록 (일별 통계·약점·아카이브된 세트) ← 과거는 전부 여기
        └─ (예정) 8주차: T4 대시보드로 확장 — 히트맵·속도 추이 그래프
```

## 데이터 흐름 (마크다운/JSON이 DB)

```
학습자 드릴 → POST /api/answer
  ├─ review-state.json  (항목별 SRS 박스·다음 복습일)  → "복습 대기" 카드의 원천
  └─ session-log.jsonl  (전체 답안 + 응답속도 ms)      → 현황·기록·튜터 피드백의 원천

세트 라이프사이클: 튜터가 sets.json 생성 → 학습자 완료 버튼 → 튜터 피드백 → sets-archive.json
학습지: 튜터가 today.json 교체 (전날 것은 아카이브 불필요 — 단어는 덱에, 글자는 학습표에 이미 있음)
```

## 한 달 스케일 체크리스트

- 홈이 길어지지 않는가? → 세트는 아카이브로 빠지고, 복습 대기는 "개수 + 버튼"만 표시 ✓
- 옛 내용이 잊히지 않는가? → SRS 복습 대기열 + 매일 세트 ④(누적 종합) 이중 안전망 ✓
- 새 학습 단계(가타카나→문법→문장)? → 덱 JSON + 학습지 JSON 추가로 해결, UI 불변 ✓
- 동기부여? → 현황 3숫자(즉시) + history(일별) + T4 대시보드(장기, 예정) ✓

## 서버 API (stdlib http.server, 포트 8765)

| 경로 | 용도 |
|---|---|
| GET /api/decks, /api/deck/{id} | 덱 목록·내용 |
| GET /api/state | SRS 상태 (복습 대기 계산용) |
| GET /api/stats | 현황 + 일별 통계 |
| GET /api/sets, POST /api/sets/done | 오늘의 세트, 완료 표시 |
| GET /api/sets/archive | 아카이브된 세트 (기록 화면용) |
| POST /api/answer | 답안 기록 (SRS 갱신 + 로그) |
| GET /api/tts?text= | 발음 (가나=원어민 클립, 단어·문장=뉴럴 TTS 캐시) |
