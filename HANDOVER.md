# HANDOVER — 채팅 세션 인수인계 문서

> **Claude Code에게**: 이 문서는 claude.ai 채팅 세션에서 설계한 일본어 학습 프로젝트의 인수인계 문서다.
> 첫 세션에서 이 문서를 끝까지 읽고, 하단의 [첫 세션 체크리스트]를 수행하라.
> 이후 이 문서는 수정하지 않는다 (설계 원본으로 보존). 진행 상태는 progress.md가 담당한다.

## 1. 프로젝트 개요

- 학습자: 37세 한국인 남성 개발자. 상세 프로필과 세션 규칙은 `CLAUDE.md` 참조 (이미 작성 완료).
- 목표: ① 일본 애니/드라마/노래 이해도 향상 ② 여행 기본 회화. 시험(JLPT) 아님.
- 기간: **1년 지속** 가능한 시스템이 목표. 8주 커리큘럼(curriculum.md)은 시작점일 뿐이다.
- 이미 완성된 것: CLAUDE.md, curriculum.md(Day 1~56), progress.md, notes/vocab.md, notes/mistakes.md, content/clips.md, 커스텀 스킬 5종(/daily, /quiz, /roleplay, /analyze, /wrapup)

## 2. 시스템 설계 원칙 (변경 금지에 가까운 합의사항)

1. **마크다운 파일이 single source of truth다.** vocab.md, mistakes.md, progress.md가 데이터베이스다. 도구는 이 파일들을 읽고 쓰는 방식으로 만든다. 별도 DB 도입 금지 (지속성과 가독성이 우선).
2. **의존성 최소화.** 도구는 표준 라이브러리 Python 또는 의존성 없는 정적 HTML+JS로. 가상환경/npm 프로젝트가 필요할 만큼 커지면 학습자와 먼저 상의.
3. **필요해질 때 만든다.** 아래 백로그를 한 번에 다 만들지 말 것. 커리큘럼 진도가 그 도구를 필요로 하는 시점에 하나씩 빌드한다.
4. **학습 시간을 도구 개발이 잠식하면 안 된다.** 학습자는 개발자라 도구 만들기로 도피할 위험이 있다(본인 동의된 농담). 세션의 기본값은 항상 공부다.

## 3. 도구 빌드 백로그 (우선순위 순)

### T1. 문자 퀴즈 앱 — Day 1 시작 전 빌드
- 정적 HTML+JS 단일 파일 (`tools/kana-quiz.html`), 브라우저로 열어서 사용
- 히라가나/가타카나 → 발음 맞히기, 반대 방향, 행(行) 단위 범위 선택
- 오답 가중 반복 (간단한 라이트너 방식이면 충분)
- 결과는 localStorage 저장으로 충분 (문자 단계는 2주면 끝나므로 과설계 금지)

### T2. Anki 내보내기 스크립트 — 3주차 시작 시 빌드
- `tools/export_anki.py`: vocab.md 표를 파싱해 Anki 임포트용 CSV/TSV 생성
- 실행: `python tools/export_anki.py` → `exports/vocab_YYYYMMDD.csv`
- 중복 내보내기 방지 (마지막 내보내기 시점 기록)

### T3. SRS 단어 퀴즈 — 4~5주차쯤, vocab.md가 100단어 넘으면 빌드
- vocab.md + mistakes.md를 데이터 소스로 하는 복습 도구
- 구현 형태(로컬 HTML + 로컬 서버 vs Python CLI)는 네가 판단해서 제안하되, 복습 이력이 파일(JSON)로 남아야 함
- SM-2 수준의 간단한 간격 반복이면 충분

### T4. 진도 대시보드 — 8주차 완주 시점에 빌드
- progress.md, vocab.md, mistakes.md를 파싱해 통계 시각화: 연속 학습일, 누적 단어 수, 오답 해결률, 주간 학습 패턴
- `tools/dashboard.py` 실행 → HTML 리포트 생성 (브라우저로 열기)
- 목적: 1년 지속의 동기부여 장치. 숫자가 쌓이는 게 보여야 한다.

### T5. 콘텐츠 어휘 추출기 — 8주 이후, clips.md가 쌓이면 빌드
- clips.md에 축적된 대사/가사에서 빈출 단어·표현 추출, vocab.md와 대조해 미학습 단어 리포트
- "다음에 뭘 외울까"를 실제 소비하는 콘텐츠 기반으로 결정하는 도구

## 4. 1년 로드맵

| 구간 | 내용 | 도구 |
|---|---|---|
| 1~2개월 | curriculum.md의 8주 과정 (문자→문법 코어→여행회화→콘텐츠 진입) | T1, T2, T3 |
| 3~6개월 | /analyze 중심의 콘텐츠 몰입기. 주 5회 클립 분석 + 주 1회 /roleplay + 매일 /quiz. 필요 문법(수동·사역·조건·경어 기초)은 콘텐츠에서 나올 때 역학습 | T4 |
| 7~12개월 | 유지·확장기. 받아쓰기 난이도 상승(자막 없이 듣고 쓰기), 좋아하는 작품 1개 "정주행 분석" 프로젝트, 여행 예정 시 회화 집중 부트캠프 2주 | T5 |

- 월 1회, 매달 첫 세션에서 지난달 회고를 progress.md에 남기고 커리큘럼/도구 우선순위를 재조정한다 (네가 먼저 제안할 것).
- 3개월 차에 8주 커리큘럼 이후의 상세 계획이 없다는 걸 잊지 말 것. 그 시점 학습자의 실력과 소비 콘텐츠 기반으로 다음 분기 계획을 함께 짜라.

## 5. 채팅 세션에서 나온 기타 합의

- 설명은 한국어 화자 이점(어순·한자어·조사 대응)을 적극 활용
- 예문·퀴즈는 교과서체보다 실제 구어/애니 표현 우선, 6주차부터는 반말(보통형) 비중 확대
- 문제는 반드시 한 번에 하나씩 (쏟아내기 금지), 모든 학습자 발화는 첨삭
- 넷플릭스 + Language Reactor 확장을 병행 몰입 도구로 사용 중이라는 전제
- 롤플레이 시나리오 목록과 출제 배분 규칙은 각 스킬 파일에 이미 정의됨

## 6. 첫 세션 체크리스트

1. CLAUDE.md, curriculum.md, 스킬 5종을 읽고 구조 파악 (수정 필요해 보이는 점이 있으면 변경 전에 학습자에게 제안)
2. T1(문자 퀴즈 앱) 빌드 → 학습자가 브라우저에서 열어 동작 확인
3. /daily로 Day 1 수업 시작 (히라가나 あ행·か행)
4. 세션 종료 시 /wrapup으로 progress.md 첫 기록 작성
